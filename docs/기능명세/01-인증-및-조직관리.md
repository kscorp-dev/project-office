# 기능 명세서: 인증/인가 및 조직도/인사관리

## 1. 인증/인가

### 1.1 로그인

**기능 설명:** 이메일과 비밀번호를 사용한 사용자 인증

**사용자 시나리오:**
1. 사용자가 이메일과 비밀번호 입력
2. "로그인" 버튼 클릭
3. 인증 성공 시 메인 대시보드로 이동
4. 인증 실패 시 에러 메시지 표시

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| AUTH-001 | 이메일 + 비밀번호 로그인 | 필수 |
| AUTH-002 | 로그인 상태 유지 (Remember Me) | 필수 |
| AUTH-003 | 로그인 실패 시 5회 제한 후 계정 잠금 (30분) | 필수 |
| AUTH-004 | 자동 로그아웃 (세션 만료: 웹 30분 비활동, 모바일 7일) | 필수 |
| AUTH-005 | 다중 기기 동시 로그인 지원 (최대 3기기) | 필수 |
| AUTH-006 | 로그인 이력 조회 (IP, 시간, 기기) | 중요 |
| AUTH-007 | 비밀번호 변경 (현재 비밀번호 확인 필요) | 필수 |
| AUTH-008 | 비밀번호 찾기 (이메일 인증) | 필수 |
| AUTH-009 | 비밀번호 정책: 8자 이상, 영문+숫자+특수문자 | 필수 |
| AUTH-010 | 비밀번호 90일 주기 변경 권고 알림 | 선택 |

**입력 유효성 검사:**
- 이메일: 유효한 이메일 형식
- 비밀번호: 최소 8자, 최대 128자
- 빈 값 제출 방지

---

### 1.2 회원가입 (관리자 초대 기반)

**기능 설명:** 시스템 관리자가 사용자를 초대하고, 초대받은 사용자가 계정을 활성화

**사용자 시나리오:**
1. 관리자가 사용자 이메일로 초대 발송
2. 사용자가 초대 이메일의 링크 클릭
3. 비밀번호 설정 및 기본 정보 입력
4. 계정 활성화 완료

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| REG-001 | 관리자 초대 기반 가입 (자유 가입 불가) | 필수 |
| REG-002 | 초대 링크 유효기간 48시간 | 필수 |
| REG-003 | 초대 시 부서/직급 사전 설정 | 필수 |
| REG-004 | 초기 비밀번호 설정 | 필수 |
| REG-005 | 프로필 사진 업로드 (선택) | 중요 |
| REG-006 | 일괄 초대 (CSV 업로드) | 선택 |

---

### 1.3 프로필 관리

**기능 설명:** 사용자 개인 정보 조회 및 수정

**수정 가능 항목:**
- 프로필 사진
- 전화번호
- 비밀번호
- 알림 설정

**수정 불가 항목 (관리자만 변경):**
- 이름, 이메일, 사번
- 부서, 직급, 직책
- 역할 (권한)

---

### 1.4 권한 체계

**역할(Role) 정의:**

| 역할 | 코드 | 설명 |

## 2. 조직도/인사관리

### 2.1 조직도

**기능 설명:** 회사 조직 구조를 트리 형태로 시각화하고 관리

**사용자 시나리오 (조회):**
1. 조직도 메뉴 진입
2. 회사 전체 트리 구조 표시
3. 부서 클릭 시 하위 부서 및 소속 직원 표시
4. 직원 클릭 시 상세 정보 및 빠른 액션 (메신저, 메일, 전화)

**사용자 시나리오 (관리 - 관리자):**
1. 부서 추가/수정/삭제/이동
2. 직원 부서 배치/이동
3. 직급/직책 변경
4. 겸직 설정

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| ORG-001 | 트리 구조 조직도 표시 (최대 5뎁스) | 필수 |
| ORG-002 | 부서별 소속 직원 목록 표시 | 필수 |
| ORG-003 | 직원 프로필 카드 (이름, 직급, 연락처, 프로필 사진) | 필수 |
| ORG-004 | 직원 이름/부서 검색 | 필수 |
| ORG-005 | 조직도에서 바로 메신저 대화 시작 | 필수 |
| ORG-006 | 조직도에서 바로 전화 연결 (모바일) | 중요 |
| ORG-007 | 부서 CRUD (관리자) | 필수 |
| ORG-008 | 부서 순서 드래그&드롭 정렬 | 중요 |
| ORG-009 | 직원 부서 이동 (관리자) | 필수 |
| ORG-010 | 겸직 설정 (다중 부서 소속) | 선택 |
| ORG-011 | 부서장 지정 | 필수 |
| ORG-012 | 조직도 인쇄/PDF 내보내기 | 선택 |

---

### 2.2 인사 관리

**기능 설명:** 직원 정보의 등록, 수정, 관리

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| HR-001 | 직원 등록 (초대) | 필수 |
| HR-002 | 직원 정보 수정 (부서, 직급, 직책, 연락처) | 필수 |
| HR-003 | 직원 상태 변경 (재직, 휴직, 퇴직) | 필수 |
| HR-004 | 직원 목록 조회 (필터: 부서, 직급, 상태) | 필수 |
| HR-005 | 직원 상세 정보 조회 | 필수 |
| HR-006 | 인사 발령 처리 (부서 이동, 직급 변경) | 중요 |
| HR-007 | 인사 발령 이력 조회 | 중요 |
| HR-008 | 직원 일괄 등록 (CSV) | 선택 |
| HR-009 | 퇴직 처리 (계정 비활성화, 데이터 보관) | 필수 |

**직급 체계:**
```
인턴 → 사원 → 주임 → 대리 → 과장 → 차장 → 부장 → 이사 → 상무 → 전무 → 부사장 → 사장
```

**직책 체계:**
```
팀원 → 파트장 → 팀장 → 부서장 → 본부장 → 대표
```

## 3. 비즈니스 규칙

### 3.1 사용자 관련
- 이메일은 시스템 전체에서 고유해야 한다
- 사번은 시스템 전체에서 고유해야 한다
- 탈퇴/퇴직 시 계정은 soft delete 처리한다 (데이터 보관)
- 퇴직자의 결재 문서, 게시글 등은 그대로 유지한다
- 퇴직자가 결재선에 있는 경우, 해당 결재선은 자동으로 건너뛴다

### 3.2 조직 관련
- 모든 활성 사용자는 반드시 1개 이상의 부서에 소속되어야 한다
- 부서 삭제 시 소속 직원이 있으면 삭제 불가 (선 이동 필요)
- 부서장은 해당 부서의 dept_admin 역할을 자동 부여받는다
- 최상위 부서(회사)는 삭제할 수 없다
