# 기능 명세서: CCTV 모니터링

## 1. 개요

회사가 보유한 CCTV 카메라의 실시간 영상을 웹과 모바일에서 모니터링하는 기능이다.
RTSP 프로토콜의 카메라 스트림을 HLS로 변환하여 모든 플랫폼에서 재생 가능하게 한다.

---

| LIVE-008 | 현재 시간 오버레이 표시 | 필수 |
| LIVE-009 | 카메라 상태 표시 (온라인 🟢 / 오프라인 🔴) | 필수 |
| LIVE-010 | 영상 품질 선택 (고화질/저화질) | 선택 |

### 2.3 PTZ 카메라 제어

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| PTZ-001 | 상하좌우 이동 (Pan/Tilt) 버튼 제어 | 중요 |
| PTZ-002 | 줌 인/아웃 (Zoom) | 중요 |
| PTZ-003 | PTZ 프리셋 위치 저장/이동 | 선택 |
| PTZ-004 | PTZ 제어 권한 체크 | 필수 |
| PTZ-005 | 동시 PTZ 제어 충돌 방지 (Lock 방식) | 중요 |

**PTZ 제어 충돌 방지 규칙:**
- PTZ 제어 시 해당 카메라에 Lock 설정 (최대 60초)
- Lock 보유자만 제어 가능
- Lock 만료 시 자동 해제
- 관리자는 Lock 강제 해제 가능

## 3. 스트리밍 기술 상세

### 3.1 RTSP → HLS 변환 프로세스

```
CCTV Camera (RTSP)
     │
     ▼
FFmpeg Process (1개 카메라당 1개 프로세스)
     │
     ├── Input: rtsp://camera_ip:554/stream
     │
     ├── 변환 설정:
     │   - 코덱: H.264 (copy 또는 재인코딩)
     │   - 세그먼트 길이: 2초
     │   - 플레이리스트: 최근 5개 세그먼트 유지
     │   - 해상도: 원본 유지 (또는 720p 다운스케일)
     │
     └── Output: /hls/{camera_id}/
         ├── stream.m3u8 (HLS 플레이리스트)
         ├── segment_001.ts
         ├── segment_002.ts
         └── ...
```

### 3.2 FFmpeg 명령어 예시

```bash
ffmpeg -rtsp_transport tcp -i rtsp://camera_ip:554/stream \
  -c:v copy -an \
  -f hls \
  -hls_time 2 \
  -hls_list_size 5 \
  -hls_flags delete_segments+append_list \
  -hls_segment_filename '/hls/{camera_id}/segment_%03d.ts' \
  /hls/{camera_id}/stream.m3u8
```

### 3.3 녹화 프로세스

```
FFmpeg (스트리밍 프로세스와 별도)
     │
     ├── Input: rtsp://camera_ip:554/stream
     │
     ├── 녹화 설정:
     │   - 파일 분할: 1시간 단위
     │   - 포맷: MP4 (H.264)
     │   - 파일명: {camera_id}_{YYYYMMDD}_{HHmmss}.mp4
     │
     └── Output: /recordings/{camera_id}/{date}/
         ├── cam01_20260408_090000.mp4
         ├── cam01_20260408_100000.mp4
         └── ...
```

| 플랫폼 | 재생 기술 | 비고 |

## 5. 권한 체계

| 권한 | 설명 | 기본 대상 |
|------|------|----------|
| can_view | 실시간 영상 시청 | 부여된 사용자/부서 |

## 6. 비즈니스 규칙

1. 카메라 접근 권한이 없는 사용자에게는 카메라 목록 자체가 표시되지 않는다
2. PTZ 제어는 한 번에 1명만 가능하다 (Lock 기반)
3. 스트리밍 시작 시 서버 부하를 고려하여 동시 시청자 수를 제한할 수 있다 (카메라당 최대 20명)
4. 녹화 영상은 설정된 보관 기간이 지나면 자동 삭제된다
5. 카메라 연결 끊김 시 3회 자동 재연결을 시도한다 (10초, 30초, 60초 간격)
6. RTSP URL은 관리자만 조회/수정할 수 있다 (보안)
7. 스냅샷 캡처 시 시간/카메라명 워터마크를 포함한다
8. 모바일에서는 데이터 절약을 위해 기본 저화질 스트리밍, 수동 고화질 전환
