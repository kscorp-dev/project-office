# 12. 모바일 앱 풀스택 기획 (iOS + Android)

> 작성: 2026-04-23 · 대상 버전: **v0.20.x → v1.0 (모바일 정식 출시)**
>
> 범위: 현재 PC(웹)판의 **13개 모듈 전체**를 iOS·Android 네이티브 수준 UX로 포팅 + 모바일 고유 기능(푸시/GPS/생체/카메라/캘린더 통합) 추가

---

## 0. 요약 (Executive Summary)

- **기술 전략**: 이미 구축된 **Expo SDK 52 (React Native 0.76)** 기반을 유지하고, 필요한 부분만 **Expo Modules API + Bare Workflow** 로 네이티브 모듈을 직접 작성한다. 순수 Swift/Kotlin 2벌 개발은 **권장하지 않음** (공수 3-5배, 이미 자산 있음, iOS/Android 간 기능 편차 관리 지옥).
- **제품 포지셔닝**: 모바일은 "출장 중·현장에서의 **결정 지원 + 긴급 대응**" 도구. PC는 "**깊은 작업**" 도구. 기능 중복 ≠ UX 중복.
- **마일스톤**: Phase 0 (토대 보강, 2주) → Phase 1 (핵심 4 기능, 4주) → Phase 2 (확장 7 기능, 6주) → Phase 3 (모바일 고유 기능, 3주) → Phase 4 (스토어 심사 + Pilot, 2주). **총 17주 (약 4 개월)** 목표.
- **최소 성공 기준 (v1.0 GA)**: 전자결재 승인/반려 · 메신저 · 메일 확인 · 푸시 알림 · 출퇴근 GPS · 캘린더 열람 · 화상회의 참가 — **8개 "킬러 시나리오"** 완성.

## 1. 범위와 원칙

### 1.1 대상 플랫폼
- **iOS 15+** (iPhone / iPad 모두 지원. iPad 는 별도 레이아웃 최적화 **X**, 호환만)
- **Android 10+ (API 29+)** (주력 기종 상정: Samsung / Google Pixel. 저사양 기기 성능 기준 RAM 4GB)

### 1.2 원칙
1. **PC 기능 1:1 포팅 ≠ 목표**. PC에서 쓰기 어렵거나 모바일에서 자주 쓰이지 않는 기능은 **뷰어 모드**만 제공하고 편집은 PC로 유도.
2. **네이티브 UX 우선**: iOS HIG · Material Design 3 룩&필을 플랫폼별로 분기. 공통은 스타일만 다르고 코드는 재사용.
3. **오프라인 부분 지원**: 리스트/캐시는 SQLite(Drizzle) 로컬 저장. 작성/수정은 네트워크 연결 시에만 허용 (MVP 기준). 충돌 해소 UX 도 Phase 2 이후.
4. **보안 기본값**: 모든 토큰은 **Keychain (iOS) / EncryptedSharedPreferences (Android)** 에 저장. 앱 진입 시 **생체 인증** 옵션 기본 ON.
5. **스토리지 사용 최소화**: CCTV 영상·대용량 문서 등은 **다운로드 대신 스트리밍**.

### 1.3 명시적 비-목표
- 아이패드·안드로이드 태블릿 전용 멀티컬럼 레이아웃 (호환만, 최적화 X)
- 웨어러블(Apple Watch / Wear OS) 연동 (장기 과제)
- 오프라인 편집·양방향 싱크 엔진 (v1.5+)

## 2. 전략적 선택: 순수 네이티브 vs Expo/RN

### 2.1 옵션 비교

| 기준 | A. Expo/RN (현재) | B. Swift + Kotlin 2벌 개발 |
|------|-------------------|-----------------------------|
| 코드 재사용 | 약 85% 공유 | 0% (UI/로직 2벌) |
| 초기 구축 공수 | ✅ 완료 (v0.19.0) | 🔴 4-6 개월 재작성 |
| 팀 인원 | 1명 (풀스택) 충분 | iOS 전담 + Android 전담 (2명) |
| 성능 (UI 60fps) | 🟡 RN Fabric+Hermes 로 충분 | 🟢 최상 |
| 네이티브 API 접근 | Expo Modules / 직접 브릿지 가능 | 🟢 직접 |
| 스토어 심사 | 동일 (네이티브 빌드로 나옴) | 동일 |
| 빌드·배포 | EAS Build 원클릭 | Xcode / Android Studio 개별 |
| 업데이트 속도 | EAS Update (OTA) 빠름 | 스토어 심사 필수 |

### 2.2 결론: **Expo/RN 유지 + 네이티브 모듈 필요 시 직접 작성**

- 이미 `@config-plugins/react-native-webrtc` 로 화상회의 네이티브 연동 성공.
- 추후 특정 기능이 성능/API 한계에 부딪히면 **bare workflow eject 없이** Expo Modules API 로 네이티브 Swift/Kotlin 브릿지 작성 (Expo prebuild 사용).
- 최종 산출물은 **네이티브 바이너리 (ipa/aab)** 로, 스토어 등록·심사에 차이 없음. 사용자는 "네이티브 앱"으로 인지.

### 2.3 Expo SDK 유지 근거
- **v52** (RN 0.76 + New Arch Fabric)로 이미 성능 병목 해소됨.
- `expo-notifications / expo-camera / expo-calendar / expo-local-authentication / expo-secure-store / expo-location / expo-av` 전부 이미 package.json 에 포함 + app.json 퍼미션 선언됨.

### 3.1 이미 구축된 것 (v0.19.0)

### 3.2 미구현/표면적 구현

- **messenger**: 방 목록만. 채팅방 내부 스크롤/입력/업로드 없음
- **mail**: 리스트만. 본문 열람/작성/첨부 없음
- **기타 11화면**: 대부분 "읽기 전용 리스트"
- **푸시 알림**: expo-notifications 설치만. 실제 토큰 등록은 `POST /notifications/devices` 백엔드 있으나 모바일 UI 호출 없음
- **오프라인 캐시·SQLite**: 없음
- **딥링크 / 유니버설 링크**: 없음
- **앱 아이콘·스플래시**: placeholder 수준

## 4. PC → 모바일 기능 매핑 (13 모듈)

각 모듈은 **"모바일 지원 수준"** × **"우선순위"** 로 분류. 모바일에서 ❌ 인 건 웹으로 유도 (외부 링크 또는 "웹에서 진행" 안내).

| # | 모듈 | 리스트/조회 | 작성/수정 | 우선순위 | 모바일 특수 기능 |
|---|------|-------------|-----------|----------|-------------------|
| 1 | **전자결재** | ✅ 필수 | ✅ 필수 (승인/반려/회수) | **P0** | 푸시 + 생체인증 기반 1탭 승인 |
| 2 | **메신저** | ✅ 필수 | ✅ 필수 (채팅/첨부) | **P0** | 푸시 + 빠른 답장 (notification action) |
| 3 | **메일** | ✅ 필수 | 🟡 답장/전달만 (신규 작성은 PC) | **P1** | 푸시 (새 메일) |
| 4 | **캘린더** | ✅ 필수 | 🟡 간단 일정 추가만 (반복/참석자는 PC) | **P1** | OS 캘린더 앱 동기화 (이미 완료) + 알림 |
| 5 | **근태(출퇴근)** | ✅ | ✅ 필수 (체크인/아웃) | **P0** | **GPS 지오펜스** · QR 코드 체크인 |
| 6 | **화상회의** | ✅ | ✅ (호스트/참가) | **P1** | CallKit (iOS) / ConnectionService (Android) 통합 · 배경 블러 |
| 7 | **CCTV** | ✅ 뷰어만 | ❌ (권한 편집은 PC) | **P2** | HLS 스트리밍 + PTZ 스와이프 제어 |
| 8 | **주차관리** | ✅ | ❌ (구역 편집 PC) | **P2** | 현재 내 차 위치 확인 · 입차 알림 |
| 9 | **게시판** | ✅ | 🟡 댓글만 (글 작성은 PC) | **P2** | — |
| 10 | **작업지시서** | ✅ | 🟡 체크리스트 완료 + 댓글 + 사진 업로드 | **P1** | **카메라 현장 사진** 즉시 첨부 |
| 11 | **자재관리** | ✅ | 🟡 입출고 수량 입력 | **P2** | **바코드 스캔** 으로 품목 조회 |
| 12 | **문서관리** | ✅ 뷰어 | ❌ (업로드는 PC) | **P2** | 오프라인 다운로드 캐시 |
| 13 | **관리콘솔** | ✅ 뷰어 (모듈 토글/사용자 상태) | 🟡 제한 | **P2** | Push로 장애 알림 수신 |

### 4.1 우선순위 정의
- **P0 (Day 1 필수)**: 앱을 깔자마자 쓰는 기능. 없으면 출시 불가.
- **P1 (GA 포함)**: v1.0 정식 출시 시 포함.
- **P2 (Post-GA)**: v1.1+ 에서 순차 확장.

## 5. 모바일 특화 요구사항

### 5.1 푸시 알림 (FCM + APNs via Expo)
- **아키텍처**: 로그인 성공 시 `expo-notifications` 로 토큰 취득 → `POST /notifications/devices` 로 서버에 등록 (이미 라우트 존재)
- **전송 채널**:
  - 결재 도착 · 승인됨 · 반려됨
  - 메신저 새 메시지 · 멘션
  - 메일 새 수신
  - 캘린더 일정 N분 전 알림
  - 출퇴근 체크 알림 (시간 기반)
  - 화상회의 초대
  - (관리자) 시스템 장애 · 고위험 감사로그
- **Notification Action 버튼** (iOS/Android 공통):
  - 결재: `승인` `반려` → 서버 호출 (생체 인증 확인 후)
  - 메신저: `빠른 답장` → 텍스트 인라인 입력
  - 화상회의: `참가` → 앱 foreground + 방 입장
- **Silent Push + Badge 카운트**: 탭별 뱃지 (메일/메신저/결재) 자동 증감

### 5.2 생체 인증
- **expo-local-authentication** (Face ID / Touch ID / 지문 / 얼굴)
- 사용처:
  - 앱 진입 시 (선택, 기본 ON)
  - 결재 승인/반려 액션 직전
  - 민감 작업 (메일 발송, 비밀번호 변경, 관리콘솔 진입)
- 실패 3회 시 비밀번호 재입력 요구

### 5.3 GPS 지오펜스 (출퇴근)
- 사무실 좌표 등록 (회사 설정 → "출근 인정 반경 200m" 등)
- 앱에서 체크인 시 `expo-location` 으로 현재 위치 취득 → 서버에서 지오펜스 검증 (이미 lat/lng 필드 있음)
- 백그라운드 지오펜스는 **추후** (배터리 이슈 + Android 13 제약)

### 5.4 카메라 / 사진 첨부
- **작업지시서**: 현장 사진 바로 찍어 업로드 (이미 `expo-camera` 설치됨)
- **바코드/QR**: 자재관리 품목 조회, 출퇴근 QR 체크인
- **화상회의**: 이미 webrtc 연동됨

### 5.5 캘린더 네이티브 통합
- 이미 구현: `expo-calendar` + `/api/calendar-sync/feed/:token.ics` → OS 캘린더 앱에 ICS 구독 자동 추가 (v0.16.0 Phase 1)
- 보완: 일정 상세 딥링크로 앱 내 상세 화면 이동

### 5.6 화상회의 네이티브 호출 UI
- **iOS CallKit**: 회의 초대 도착 → "수신 전화"처럼 잠금화면에 표시
- **Android ConnectionService**: 동일
- 구현: `react-native-callkeep` 라이브러리 (Expo Config Plugin 필요)

### 5.7 오프라인 전략 (Phase 2)
- **SQLite (Drizzle ORM)** 로 아래 데이터 캐시:
  - 메신저 최근 100 메시지/방
  - 메일 INBOX 최근 50건
  - 캘린더 앞뒤 30일 일정
- 네트워크 복구 시 서버 데이터로 merge (updatedAt 기준)

### 5.8 딥링크 / 유니버설 링크
- Scheme: `projectoffice://approval/:id` 등
- iOS Associated Domains + Android App Links → `https://app.kscorp.co.kr/approval/:id` → 앱 열기
- 푸시 알림 페이로드에 `{ path: '/approval/123' }` 포함 → 탭 시 해당 화면 바로 이동

### 6.2 상태 관리
- **Zustand** (이미 채택) — 전역 상태 (auth / push / settings)
- **React Query (TanStack)** 도입 권장 — 서버 데이터 캐싱 · 재시도 · background refetch
  - 이유: 현재 모바일 화면마다 `useState + useEffect + fetch` 중복. React Query 로 **캐시 무효화 / 동시 요청 취합 / 화면 간 공유** 간소화

### 6.3 실시간 (Socket.IO)
- 이미 `socket.io-client` 설치됨
- 연결 라이프사이클:
  - 로그인 성공 시 연결
  - 백그라운드 전환 시 일시 중단 (네트워크 절약)
  - 포그라운드 복귀 + 토큰 유효 시 재연결
  - 서버는 user-specific room 에 조인 후 `notification` / `mail-new` / `chat-message` 이벤트 push

### 6.4 네트워크 / API 규약
- 모든 요청 공통 헤더:
  - `Authorization: Bearer <access_token>` (axios 인터셉터)
  - `X-Client: mobile` `X-App-Version: x.y.z` (서버 통계/호환성 체크용)
- 토큰 만료 시 자동 refresh (이미 구현)
- 에러 응답 표준: `{ success: false, error: { code, message } }` (백엔드 errorHandler 와 이미 일치)

## 7. UX 원칙

### 7.1 네비게이션
- **하단 탭 5개** (Dashboard · Approval · Messenger · Mail · More)

### 7.2 제스처
- 리스트 아이템 좌측 스와이프 → 빠른 액션 (읽음 / 즐겨찾기 / 삭제)

### 7.3 다크 모드
- `userInterfaceStyle: automatic` (app.json) — OS 설정 따름
- `useColorScheme()` 으로 컴포넌트 내 분기
- 현재 `COLORS` 상수는 다크/라이트 공용 일부만 대응 → 다크 팔레트 확정 필요

### 7.4 접근성 (A11y)
- 모든 버튼에 `accessibilityLabel`
- 최소 터치 타겟 44×44pt
- 텍스트 크기 동적 조정 (iOS Dynamic Type, Android fontScale)
- VoiceOver / TalkBack 기본 동작 확인

### 7.5 Splash / Onboarding
- 스플래시: 로고 + 회사색
- 최초 실행 3단계 온보딩:
  1. "PC로 시작한 결재, 이제 모바일로"
  2. "푸시 알림 허용 (필수)"
  3. "생체 인증 설정 (권장)"

## 8. 화면 스펙 (Day 1 킬러 시나리오 8종)

### 8.1 로그인 + 생체 등록
- 사번/비밀번호 1회 로그인 → 성공 시 "이 기기에서 Face ID로 빠른 로그인 사용" 옵션 (기본 체크)
- 이후 앱 진입 시 FaceID/지문만 통과하면 자동 로그인

### 8.2 대시보드 (상단 탭)
- 카드 4-6개: 오늘 결재 N건 / 새 메시지 N / 새 메일 N / 오늘 일정 N / 미체크인 / (관리자) 알림
- 각 카드 탭 → 해당 탭으로 이동

### 8.3 결재 리스트 → 상세 → 승인/반려
- 리스트: 기존 대기/승인/반려/내기안 4탭 (이미 구현)
- 상세: 문서 본문 + 결재선 다이어그램 + 첨부파일 미리보기
- 액션: 하단 고정 "승인" / "반려" 버튼 → **생체 인증** 통과 후 API 호출
- 푸시 도착 → 탭 → 해당 문서 상세 바로 열림

### 8.4 메신저 방 내부
- 상단: 참가자 아바타 + 방 이름
- 리스트: 말풍선 (내/상대 색 구분, 시간 · 읽음 표시)
- 하단: 입력창 + 첨부(사진/파일) + 전송
- 실시간: Socket.IO 로 새 메시지 수신
- Notification Action "빠른 답장"과도 연동

### 8.5 메일 본문 열람
- HTML 메일 렌더링 (react-native-render-html 또는 WebView)
- 첨부파일 다운로드 → 시스템 공유 시트 (Share Sheet)
- 답장/전달 (Phase 1 우선)

### 8.6 GPS 출근 체크인
- "출근" 버튼 탭 → 위치 취득 → 지오펜스 범위 내면 success
- 범위 밖이면 "사유 입력" 다이얼로그 + 관리자 승인 플로우

### 8.7 캘린더 (오늘 + 다가오는 일정)
- 월간 그리드 (간단) + 일자별 카드 리스트
- 탭 시 상세. 편집은 PC 유도 배너.

### 8.8 화상회의 참가
- 초대 푸시 → **CallKit/ConnectionService 수신 UI** (잠금화면)
- 응답 → 앱 foreground + 방 입장 (WebRTC)
