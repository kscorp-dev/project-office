# 기능 명세서: 전자결재

## 1. 개요

전자결재는 사내 문서의 기안, 결재, 승인/반려 프로세스를 디지털화하는 핵심 업무 기능이다.

---

## 2. 사용자 유형별 역할

| 역할 | 설명 |
|------|------|
| 기안자 | 결재 문서를 작성하고 상신하는 사용자 |
| 결재자 | 결재선에서 승인/반려 권한을 가진 사용자 |
| 합의자 | 의견을 제시하는 사용자 (승인/반려 권한 없음) |
| 참조자 | 문서를 열람만 할 수 있는 사용자 |
| 대결자 | 결재자 부재 시 대리 결재하는 사용자 |

---

## 3. 기능 상세

### 3.1 결재 문서 작성 (기안)

**사용자 시나리오:**
1. "새 결재 작성" 클릭
2. 문서 양식 선택 (품의서, 휴가신청서, 지출결의서 등)
3. 양식에 따른 필드 입력
4. 결재선 설정
5. 첨부파일 추가 (선택)
6. "임시저장" 또는 "상신" 선택

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| APR-001 | 양식 선택 후 해당 양식의 입력 폼 표시 | 필수 |
| APR-002 | 본문 에디터 (서식, 표, 이미지 삽입) | 필수 |
| APR-003 | 결재선 설정 (조직도에서 선택) | 필수 |
| APR-004 | 자주 쓰는 결재선 저장/불러오기 | 중요 |
| APR-005 | 첨부파일 (최대 5개, 개별 20MB) | 필수 |
| APR-006 | 임시저장 | 필수 |
| APR-007 | 긴급도 설정 (일반, 긴급, 초긴급) | 중요 |
| APR-008 | 문서 번호 자동 채번 (양식-연도-순번) | 필수 |
| APR-009 | 이전 문서 복사 작성 | 선택 |

---

### 3.2 결재선 설정

**결재선 구성 규칙:**

```
기안자 → [결재자1] → [결재자2] → ... → [최종결재자]
              ↑
         [합의자] (병렬)
         [참조자] (열람)
```

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| LINE-001 | 순차 결재 (직렬: 1→2→3 순서대로) | 필수 |
| LINE-002 | 병렬 결재 (동일 step의 결재자 동시 진행) | 중요 |
| LINE-003 | 합의자 추가 (의견만, 승인/반려 불가) | 필수 |
| LINE-004 | 참조자 추가 (열람만) | 필수 |
| LINE-005 | 결재선 최소 1명 이상 | 필수 |
| LINE-006 | 결재선 최대 10단계 | 필수 |
| LINE-007 | 양식별 기본 결재선 설정 (관리자) | 중요 |
| LINE-008 | 개인 결재선 프리셋 저장 (최대 10개) | 중요 |
| LINE-009 | 조직도에서 드래그&드롭으로 결재자 추가 | 중요 |

---

### 3.3 결재 처리

**사용자 시나리오 (결재자):**
1. 결재 요청 알림 수신 (푸시, 메신저, 앱 뱃지)
2. "결재 대기" 목록에서 문서 확인
3. 문서 내용 및 첨부파일 열람
4. "승인", "반려", "보류" 중 선택
5. 의견 작성 (반려 시 필수, 승인 시 선택)
6. 처리 완료

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| PROC-001 | 승인: 다음 단계로 진행 | 필수 |
| PROC-002 | 반려: 기안자에게 반환 (사유 필수) | 필수 |
| PROC-003 | 보류: 결재 일시 정지 | 중요 |
| PROC-004 | 의견 작성 (텍스트) | 필수 |
| PROC-005 | 최종 결재자 승인 시 문서 상태 '완료' | 필수 |
| PROC-006 | 결재 진행 상황 실시간 표시 | 필수 |
| PROC-007 | 결재 순서 도달 시 알림 발송 | 필수 |
| PROC-008 | 일괄 결재 (여러 문서 한번에 승인) | 선택 |

### 3.6 결재 문서함 (조회)

**문서함 분류:**

| 문서함 | 설명 | 표시 대상 |
|--------|------|----------|
| 기안 문서 | 내가 기안한 문서 | 기안자 본인 |
| 결재 대기 | 내 차례인 결재 문서 | 현재 결재자 |
| 결재 예정 | 내 차례가 아직 안 온 문서 | 다음 순서 결재자 |
| 결재 완료 | 내가 결재 처리한 문서 | 결재 참여자 |
| 참조 문서 | 참조로 포함된 문서 | 참조자 |
| 반려 문서 | 반려된 내 문서 | 기안자 |
| 임시 저장 | 아직 상신하지 않은 문서 | 기안자 |

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| BOX-001 | 문서함별 목록 조회 (페이지네이션) | 필수 |
| BOX-002 | 문서 검색 (제목, 기안자, 기간, 상태) | 필수 |
| BOX-003 | 문서 상태별 필터 | 필수 |
| BOX-004 | 문서 정렬 (날짜, 긴급도) | 필수 |
| BOX-005 | 결재 대기 건수 뱃지 표시 | 필수 |
| BOX-006 | 긴급 문서 우선 표시 (아이콘/색상 구분) | 중요 |

## 4. 결재 프로세스 플로우

```
[임시저장] ──상신──> [결재진행중]
                      │
              ┌───────┼───────┐
              │       │       │
         [승인진행]  [반려]   [보류]
              │       │       │
              │   기안자에게   결재자가
              │   반환       해제 시
              │       │     재개
              │       │       │
              │   [수정/재상신] │
              │       │       │
              ▼       └───────┘
         [최종승인]
              │
              ▼
         [결재완료]
         
※ 상신 후 첫 결재자 처리 전: [회수] 가능 → [임시저장]으로 복귀
```

## 5. 알림 규칙

| 이벤트 | 수신자 | 알림 방법 |
|--------|--------|----------|
| 문서 상신 | 첫 번째 결재자 | 푸시 + 메신저 + 앱 뱃지 |
| 결재 순서 도달 | 해당 순서 결재자 | 푸시 + 메신저 + 앱 뱃지 |
| 승인 | 기안자, 다음 결재자 | 푸시 |
| 최종 승인 | 기안자, 전체 결재선 | 푸시 |
| 반려 | 기안자 | 푸시 + 메신저 |
| 회수 | 결재선 전원 | 앱 내 알림 |
| 결재 독촉 | 미처리 결재자 | 푸시 + 메신저 |
| 위임 설정 | 수임자 | 푸시 |

---

## 6. 비즈니스 규칙

1. 기안자는 자신의 결재선에 자신을 포함할 수 없다
2. 같은 사용자를 결재선에 중복 추가할 수 없다
3. 결재선은 상신 후 변경할 수 없다 (회수 후 재작성만 가능)
4. 반려 시 사유는 필수이다
5. 결재 순서는 step 순으로 진행되며, 같은 step은 병렬 처리한다
6. 병렬 결재 시 모든 결재자가 승인해야 다음 step으로 진행한다
7. 병렬 결재 중 1명이라도 반려하면 문서 전체가 반려된다
8. 퇴직자가 결재선에 있는 경우 해당 단계는 자동 건너뛴다
9. 위임 기간 중 결재 요청은 수임자에게 표시되며, 처리 시 "대결" 표시된다
10. 문서 번호 형식: `{양식코드}-{연도4자리}-{순번5자리}` (예: APR-2026-00001)
11. 임시저장 문서는 30일 후 자동 삭제 (알림 후)
12. 휴가신청서 최종 승인 시 연차 자동 차감, 캘린더 자동 등록
