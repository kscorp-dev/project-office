# 기능 명세서: 재고관리 및 통계

## 1. 개요

회사 보유 자산/물품의 재고를 관리하고, 입출고 현황을 그래프 형태의 통계 대시보드로 시각화하는 모듈이다.
카테고리별, 기간별 재고 변동을 직관적으로 파악할 수 있다.

---

## 2. 기능 상세

### 2.1 품목(자산) 관리

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| INV-001 | 품목 등록 (품명, 카테고리, 단위, 초기 수량, 단가) | 필수 |
| INV-002 | 품목 수정/삭제 | 필수 |
| INV-003 | 품목 카테고리 관리 (대분류/중분류/소분류) | 필수 |
| INV-004 | 품목 이미지 등록 | 중요 |
| INV-005 | 품목 바코드/QR코드 (자동 생성) | 선택 |
| INV-006 | 품목 목록 조회 (검색, 필터, 정렬) | 필수 |
| INV-007 | 현재 재고 수량 실시간 표시 | 필수 |
| INV-008 | 안전재고 수량 설정 | 중요 |
| INV-009 | 안전재고 미달 시 알림 | 중요 |
| INV-010 | 품목 엑셀 일괄 등록 | 선택 |

**품목 기본 정보:**

| 필드 | 설명 | 예시 |
|------|------|------|
| 품명 | 물품 이름 | A4 용지, 모니터, 토너 |
| 카테고리 | 분류 | 사무용품 > 용지류 |
| SKU | 고유 관리번호 | OFF-PAP-001 |
| 단위 | 수량 단위 | 박스, 개, EA, SET |
| 단가 | 개당 가격 | 25,000원 |
| 현재 재고 | 보유 수량 | 150 |
| 안전재고 | 최소 보유 수량 | 50 |
| 보관 위치 | 창고/선반 위치 | 본사 3층 자재창고 A-03 |
| 공급업체 | 주요 공급처 | (주)오피스디포 |
| 비고 | 추가 메모 | |

### 2.2 입출고 관리

**입고 시나리오:**
1. "입고 등록" 진입
2. 품목 선택 (검색/스캔)
3. 입고 수량, 입고일, 공급처, 단가 입력
4. 등록 → 재고 수량 자동 증가

**출고 시나리오:**
1. "출고 등록" 진입
2. 품목 선택
3. 출고 수량, 출고일, 수령부서/담당자 입력
4. 등록 → 재고 수량 자동 차감

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| IO-001 | 입고 등록 (품목, 수량, 일자, 공급처, 단가) | 필수 |
| IO-002 | 출고 등록 (품목, 수량, 일자, 수령부서, 담당자) | 필수 |
| IO-003 | 반품 등록 (출고 취소 → 재고 복원) | 중요 |
| IO-004 | 재고 조정 (실사 후 수량 보정) | 중요 |
| IO-005 | 입출고 이력 조회 (품목별, 기간별) | 필수 |
| IO-006 | 입출고 시 재고 수량 자동 반영 | 필수 |
| IO-007 | 재고 부족 시 출고 경고 (마이너스 방지) | 필수 |
| IO-008 | 입출고 승인 워크플로우 (선택적 결재 연동) | 선택 |
| IO-009 | 입출고 엑셀 내보내기 | 중요 |
| IO-010 | 모바일에서 바코드 스캔 입출고 | 선택 |

## 3. 데이터 모델

### inventory_categories (재고 카테고리)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| name | VARCHAR(100) | 카테고리명 |
| parent_id | UUID | 상위 카테고리 (FK, NULL) |
| sort_order | INTEGER | 정렬 |

### inventory_items (품목)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| name | VARCHAR(200) | 품명 |
| sku | VARCHAR(50) | 관리번호 (UNIQUE) |
| category_id | UUID | 카테고리 (FK) |
| unit | VARCHAR(20) | 단위 (개, 박스, EA 등) |
| unit_price | DECIMAL(12,2) | 단가 |
| current_stock | INTEGER | 현재 재고 |
| safety_stock | INTEGER | 안전재고 |
| location | VARCHAR(100) | 보관 위치 |
| supplier | VARCHAR(200) | 공급업체 |
| image_url | VARCHAR(500) | 이미지 |
| barcode | VARCHAR(100) | 바코드 |
| description | TEXT | 비고 |
| is_active | BOOLEAN | 활성 여부 |

### inventory_transactions (입출고 내역)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| item_id | UUID | 품목 (FK → inventory_items) |
| type | ENUM | 유형 (in: 입고, out: 출고, adjust: 조정, return: 반품) |
| quantity | INTEGER | 수량 (양수) |
| unit_price | DECIMAL(12,2) | 거래 단가 |
| total_price | DECIMAL(14,2) | 총 금액 |
| before_stock | INTEGER | 거래 전 재고 |
| after_stock | INTEGER | 거래 후 재고 |
| supplier | VARCHAR(200) | 공급업체 (입고 시) |
| department_id | UUID | 수령 부서 (출고 시, FK) |
| receiver_id | UUID | 수령자 (출고 시, FK → users) |
| processed_by | UUID | 처리자 (FK → users) |
| transaction_date | DATE | 거래일 |
| note | TEXT | 비고 |

### inventory_audits (재고 실사)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| item_id | UUID | 품목 (FK) |
| expected_stock | INTEGER | 장부 수량 |
| actual_stock | INTEGER | 실제 수량 |
| difference | INTEGER | 차이 |
| audited_by | UUID | 실사자 (FK → users) |
| audit_date | DATE | 실사일 |
| note | TEXT | 비고 |

## 5. 비즈니스 규칙

1. 재고 수량은 음수가 될 수 없다 (출고 시 현재 재고 초과 불가)
2. 모든 입출고는 이력으로 기록되며 삭제할 수 없다 (취소/반품으로 처리)
3. 안전재고 미달 시 관리자에게 자동 알림 발송
4. 재고 조정(실사 보정)은 관리자 권한 필요
5. 입출고 내역은 최소 5년간 보관
6. 단가 변경 시 기존 입고 내역의 단가는 변경되지 않는다 (거래 시점 단가 보존)
7. 품목 삭제 시 입출고 이력이 있으면 비활성화만 가능 (soft delete)
8. 통계 데이터는 일 배치로 집계 테이블에 저장 (성능)
