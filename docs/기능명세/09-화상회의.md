# 기능 명세서: 화상회의

## 1. 개요

메신저와 연동되는 화상회의 기능이다. WebRTC 기반으로 실시간 영상/음성 통화를 지원하며,
모든 회의는 자동으로 기록(날짜, 시간, 참석자)이 DB에 저장된다.
회의 종료 시 AI 기반 자동 회의록이 생성되어, 상위 직급자가 검색 및 열람할 수 있다.

## 2. 기능 상세

### 2.1 화상회의 시작

**시작 방법:**

| 방법 | 설명 |
|------|------|
| 메신저에서 시작 | 채팅방 상단 "화상회의" 버튼 → 참여자 자동 초대 |
| 일정에서 시작 | 캘린더 회의 일정의 "회의 시작" 버튼 |
| 즉석 회의 | "새 회의" 생성 → 참여 링크 공유 |

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| MEET-001 | 메신저 채팅방에서 화상회의 시작 | 필수 |
| MEET-002 | 회의 시작 시 채팅방 멤버에게 참여 알림 | 필수 |
| MEET-003 | 즉석 회의 생성 (참여 링크 생성) | 필수 |
| MEET-004 | 회의 예약 (캘린더 연동) | 중요 |
| MEET-005 | 예약 회의 시작 시간 5분 전 알림 | 중요 |
| MEET-006 | 회의 참여 링크로 입장 | 필수 |
| MEET-007 | 최대 참여자 수 제한 (기본 20명) | 필수 |

### 2.4 자동 회의록 생성

**핵심 기능: 회의 종료 시 AI 기반 자동 회의록 생성**

**프로세스:**
```
회의 진행 중
    │
    ├── 음성 → 실시간 STT (Speech-to-Text) 변환
    │         (각 발언자별 텍스트 기록)
    │
    ▼
회의 종료
    │
    ├── 전체 발언 텍스트 수집
    │
    ├── AI 모델 (Claude API / 자체 모델)로 요약
    │   ├── 회의 주제 추출
    │   ├── 주요 논의 사항 정리
    │   ├── 결정 사항 추출
    │   ├── 액션 아이템 추출 (담당자, 기한)
    │   └── 전체 요약 생성
    │
    ▼
자동 회의록 생성 → DB 저장
    │
    ├── 참석자에게 회의록 알림
    └── 회의록 수정 가능 (호스트/참석자)
```

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| MINUTES-001 | 회의 중 실시간 음성→텍스트 변환 (STT) | 필수 |
| MINUTES-002 | 발언자별 텍스트 구분 기록 | 필수 |
| MINUTES-003 | 회의 종료 시 AI 자동 요약 생성 | 필수 |
| MINUTES-004 | 자동 회의록 구성: 주제, 논의사항, 결정사항, 액션아이템 | 필수 |
| MINUTES-005 | 회의록 수동 수정/보완 (호스트, 참석자) | 필수 |
| MINUTES-006 | 회의록 확정 (수정 완료 후 잠금) | 중요 |
| MINUTES-007 | 회의록 PDF 다운로드 | 중요 |
| MINUTES-008 | 전체 발언 원문 조회 (타임스탬프 포함) | 중요 |

**자동 회의록 템플릿:**

### 2.5 회의 검색 (상위 직급 전용)

**상위 직급(과장 이상)은 자신이 참석하지 않은 회의도 검색/열람 가능**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| SRCH-001 | 회의 기록 검색 (기간, 참석자, 키워드) | 필수 |
| SRCH-002 | 회의록 내용 전문 검색 | 필수 |
| SRCH-003 | 부서별 회의 이력 필터 | 필수 |
| SRCH-004 | 회의 통계 (월별 회의 횟수, 총 시간) | 중요 |
| SRCH-005 | 발언 원문에서 키워드 검색 | 중요 |
| SRCH-006 | 회의 녹화 영상 재생 | 중요 |

**검색 권한 체계:**

| 직급 | 검색 범위 |
|------|----------|
| 사원~대리 | 본인 참석 회의만 |
| 과장~차장 | 소속 부서 전체 회의 |
| 부장 이상 | 하위 모든 부서 회의 |
| 관리자 (admin) | 전사 모든 회의 |

## 3. 기술 아키텍처

### 3.1 WebRTC 기반 화상회의

```
┌──────────┐          ┌───────────────┐          ┌──────────┐
│ Client A │◄────────►│  SFU Server   │◄────────►│ Client B │
│          │  WebRTC  │ (mediasoup /  │  WebRTC  │          │
│          │          │  Janus)       │          │          │
└──────────┘          └───────┬───────┘          └──────────┘
                              │
                    ┌─────────┴─────────┐
                    │  Signal Server    │
                    │  (Socket.IO)      │
                    │  - Room 관리      │
                    │  - SDP 교환       │
                    │  - ICE Candidate  │
                    └───────────────────┘
```

### 3.2 STT + AI 회의록 파이프라인

```
Audio Stream (WebRTC)
    │
    ▼
STT Engine (선택)
├── Option A: Web Speech API (브라우저 내장, 무료)
├── Option B: Google Cloud STT (고정확도)
├── Option C: Whisper API (OpenAI)
└── Option D: Clova STT (네이버, 한국어 최적화)
    │
    ▼
발언 텍스트 (발언자, 타임스탬프)
    │
    ▼
AI 요약 (회의 종료 후)
├── Claude API (Anthropic)
└── 프롬프트: "다음 회의 발언록을 분석하여 
     주제, 논의사항, 결정사항, 액션아이템을 정리하세요"
    │
    ▼
회의록 생성 → DB 저장
```

## 4. 데이터 모델

### meetings (회의)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| title | VARCHAR(200) | 회의 제목 |
| host_id | UUID | 주최자 (FK → users) |
| chat_room_id | UUID | 연결 채팅방 (FK → chat_rooms, NULL) |
| status | ENUM | 상태 (scheduled, in_progress, ended, cancelled) |
| scheduled_at | TIMESTAMP | 예약 일시 (NULL이면 즉석 회의) |
| started_at | TIMESTAMP | 실제 시작 시간 |
| ended_at | TIMESTAMP | 종료 시간 |
| duration_minutes | INTEGER | 소요 시간 (분) |
| recording_url | VARCHAR(500) | 녹화 파일 URL |
| max_participants | INTEGER | 최대 참여자 수 |

### meeting_participants (회의 참석자)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| meeting_id | UUID | 회의 (FK → meetings) |
| user_id | UUID | 참석자 (FK → users) |
| role | ENUM | 역할 (host, participant) |
| joined_at | TIMESTAMP | 입장 시간 |
| left_at | TIMESTAMP | 퇴장 시간 |
| duration_minutes | INTEGER | 참여 시간 (분) |

### meeting_transcripts (발언 기록)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| meeting_id | UUID | 회의 (FK → meetings) |
| speaker_id | UUID | 발언자 (FK → users) |
| content | TEXT | 발언 텍스트 |
| timestamp_start | INTEGER | 시작 시점 (초) |

### meeting_minutes (회의록)
| 컬럼 | 타입 | 설명 |

|------|------|------|

## 5. 알림 규칙

| 이벤트 | 수신자 | 알림 |
|--------|--------|------|
| 회의 시작 | 채팅방 멤버 / 초대자 | 푸시 + 메신저 |
| 예약 회의 5분 전 | 참석자 | 푸시 |
| 회의 종료 | 참석자 | 앱 내 알림 |
| 회의록 생성 완료 | 참석자 | 푸시 |
| 회의록 수정 | 참석자 | 앱 내 알림 |
| 회의록 확정 | 참석자 + 상위 직급 | 앱 내 알림 |

## 6. 비즈니스 규칙

1. 회의 주최자(호스트)만 회의를 종료할 수 있다
2. 호스트가 퇴장하면 가장 먼저 입장한 참석자가 호스트가 된다
3. 모든 참석자가 퇴장하면 회의는 자동 종료된다
4. 회의 기록(날짜, 시간, 참석자)은 회의 종료 시 자동으로 DB에 저장된다
5. STT는 회의 시작 시 자동 활성화 (참석자에게 고지)
6. 자동 회의록은 회의 종료 후 5분 이내에 생성된다
7. 회의록 수정은 호스트와 참석자만 가능 (상위 직급은 열람만)
8. 회의록 확정 후에는 수정 불가
9. 녹화 파일은 90일간 보관 후 자동 삭제 (설정 변경 가능)
10. 회의 검색 시 상위 직급자는 하위 부서의 회의만 열람 가능 (타 본부 X)
11. 회의 중 채팅 내용도 회의 기록에 포함되어 검색 가능
