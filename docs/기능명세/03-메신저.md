# 기능 명세서: 메신저

## 1. 개요

실시간 사내 메신저로 1:1 채팅, 그룹 채팅, 파일 공유, 멘션 등의 기능을 제공한다.
WebSocket 기반으로 실시간 메시지 송수신을 지원하며, 오프라인 시 푸시 알림을 통해 메시지를 전달한다.

---

## 2. 기능 상세

### 2.1 채팅방 관리

**채팅방 유형:**

| 유형 | 설명 | 특징 |
|------|------|------|
| 1:1 채팅 (direct) | 두 사용자 간 대화 | 이름 자동 설정, 삭제 불가 |
| 그룹 채팅 (group) | 3인 이상 대화 | 이름 설정 가능, 멤버 초대/퇴장 |

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| MSG-001 | 1:1 채팅방 생성 (조직도/주소록에서 선택) | 필수 |
| MSG-002 | 그룹 채팅방 생성 (복수 사용자 선택) | 필수 |
| MSG-003 | 그룹 채팅방 이름 설정/변경 | 필수 |
| MSG-004 | 그룹 채팅방 멤버 추가 초대 | 필수 |
| MSG-005 | 그룹 채팅방 나가기 | 필수 |
| MSG-006 | 기존 1:1 대화 상대와 대화 시 기존 방 재사용 | 필수 |
| MSG-007 | 채팅방 목록 최신 메시지 순 정렬 | 필수 |
| MSG-008 | 채팅방 고정 (상단 고정, 최대 5개) | 중요 |
| MSG-009 | 채팅방별 알림 켜기/끄기 | 중요 |
| MSG-010 | 채팅방 검색 (방 이름, 참여자 이름) | 필수 |

### 2.2 메시지 전송

**지원 메시지 유형:**

| 유형 | 설명 | 최대 크기 |
|------|------|----------|
| 텍스트 | 일반 텍스트 메시지 | 5,000자 |
| 이미지 | JPG, PNG, GIF, WebP | 10MB |
| 파일 | 문서, 압축파일 등 | 50MB |
| 시스템 | 입장/퇴장/초대 알림 | 자동 생성 |

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| SEND-001 | 텍스트 메시지 전송 | 필수 |
| SEND-002 | 이미지 전송 (미리보기 썸네일 표시) | 필수 |
| SEND-003 | 파일 전송 (아이콘 + 파일명 + 크기 표시) | 필수 |
| SEND-004 | 여러 파일 동시 전송 (최대 10개) | 중요 |
| SEND-005 | 이미지 클립보드 붙여넣기 전송 | 중요 |
| SEND-006 | 드래그 앤 드롭 파일 전송 | 중요 |
| SEND-007 | 답장 (특정 메시지에 대한 회신) | 중요 |
| SEND-008 | 메시지 수정 (발송 후 5분 이내) | 중요 |
| SEND-009 | 메시지 삭제 (본인 메시지만) | 필수 |
| SEND-010 | 이모지 입력 (이모지 피커) | 중요 |
| SEND-011 | 엔터 키로 전송, Shift+Enter로 줄바꿈 | 필수 |
| SEND-012 | URL 자동 링크 변환 | 중요 |
| SEND-013 | URL 미리보기 (OG 메타데이터) | 선택 |

### 2.4 읽음 확인

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| READ-001 | 1:1 채팅: 읽음/안읽음 표시 (체크 마크) | 필수 |
| READ-002 | 그룹 채팅: 안 읽은 인원 수 표시 | 필수 |
| READ-003 | 안 읽은 인원 수 클릭 시 읽은/안읽은 사람 목록 | 중요 |
| READ-004 | 채팅방 목록에서 안 읽은 메시지 수 뱃지 | 필수 |
| READ-005 | 전체 안 읽은 메시지 수 앱 뱃지 | 필수 |

**읽음 처리 규칙:**
- 채팅방에 진입하면 해당 시점까지의 모든 메시지를 읽음 처리
- 채팅방이 열린 상태에서 수신되는 메시지는 즉시 읽음 처리
- 읽음 상태는 각 멤버의 `last_read_message_id`로 판단

## 3. 메시지 동기화

### 3.1 초기 로딩
- 채팅방 진입 시 최근 50개 메시지 로딩
- 위로 스크롤 시 이전 메시지 20개씩 추가 로딩 (무한 스크롤)

### 3.2 실시간 수신
- WebSocket 연결 상태에서 새 메시지 즉시 표시
- 연결 끊김 시 자동 재연결 (1초, 2초, 4초... 최대 30초 간격)
- 재연결 시 마지막 수신 메시지 이후 메시지 동기화

### 3.3 오프라인 처리
- 오프라인 시 전송된 메시지는 로컬에 저장
- 온라인 복귀 시 자동 전송 (전송 실패 시 재시도)
- 오프라인 동안 수신된 메시지는 재연결 시 서버에서 가져옴

## 4. 채팅방 데이터 관리

### 4.1 메시지 보관
- 메시지는 서버에 영구 보관
- 삭제된 메시지는 "삭제된 메시지입니다" 표시 (soft delete)
- 수정된 메시지는 "(수정됨)" 표시 + 수정 시간

### 4.2 파일 보관
- 전송된 파일은 서버에 영구 보관
- 첨부파일 다운로드 시 원본 파일명으로 저장

## 5. 비즈니스 규칙

1. 1:1 채팅방은 동일 상대와 중복 생성 불가 (기존 방 재사용)
2. 그룹 채팅방 생성자가 자동으로 방장이 된다
3. 방장이 나가면 가장 먼저 입장한 멤버가 방장이 된다
4. 그룹 채팅방 멤버가 1명이 되면 채팅방은 비활성 처리
5. 퇴장한 사용자는 이전 대화 내역을 볼 수 없다
6. 새로 초대된 사용자는 초대 시점 이후의 메시지만 볼 수 있다
7. 메시지 수정은 발송 후 5분 이내만 가능하다
8. 파일 전송 시 확장자 제한: exe, bat, cmd, sh, ps1 등 실행 파일 차단
9. 탈퇴/퇴직 사용자의 메시지는 그대로 유지하되, "탈퇴한 사용자" 표시
10. 시스템 메시지 (입장/퇴장/초대)는 삭제/수정할 수 없다
