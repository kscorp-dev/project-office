# 기능 명세서: 캘린더 / 게시판

---

# Part 1: 캘린더

## 1. 개요

개인 및 공유 일정을 관리하고, 전자결재(휴가), 근태, 회의 등과 자동 연동되는 통합 일정 관리 기능이다.

---

## 2. 기능 상세

### 2.1 일정 관리

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| CAL-001 | 일정 등록 (제목, 일시, 장소, 설명) | 필수 |
| CAL-002 | 일정 수정/삭제 | 필수 |
| CAL-003 | 종일 일정 지원 | 필수 |
| CAL-004 | 반복 일정 (매일, 매주, 매월, 매년, 커스텀) | 중요 |
| CAL-005 | 일정 색상 구분 | 중요 |
| CAL-006 | 일정 알림 (5분/15분/30분/1시간/1일 전) | 필수 |
| CAL-007 | 참석자 초대 및 수락/거절 | 중요 |
| CAL-008 | 일/주/월 보기 전환 | 필수 |
| CAL-009 | 일정 드래그&드롭 이동 (웹) | 중요 |
| CAL-010 | 일정 검색 (제목, 기간) | 필수 |

### 2.2 캘린더 유형

| 유형 | 설명 | 권한 |
|------|------|------|
| 개인 캘린더 | 본인만 보는 일정 | 본인만 관리 |
| 부서 캘린더 | 부서 공유 일정 | 부서원 조회, 부서장 관리 |
| 전사 캘린더 | 회사 전체 일정 (공휴일, 행사) | 관리자만 등록 |
| 연동 캘린더 | 휴가/근태 자동 표시 | 시스템 자동 (읽기전용) |

### 2.3 타 모듈 연동

| 연동 모듈 | 연동 내용 | 동작 |
|----------|----------|------|
| 휴가 | 승인된 휴가 | 캘린더에 자동 등록 (해당일 "연차" 표시) |
| 출장 | 승인된 출장 | 캘린더에 자동 등록 |
| 화상회의 | 예약된 회의 | 캘린더에 자동 등록 (참여 링크 포함) |

---

## 3. 비즈니스 규칙

1. 연동 캘린더의 일정은 수동 수정/삭제 불가 (원본 모듈에서만 변경)
2. 반복 일정 수정 시 "이 일정만 / 이후 모든 일정 / 모든 일정" 선택
3. 전사 캘린더는 관리자만 일정 추가 가능
4. 삭제된 일정은 30일간 휴지통에 보관 후 영구 삭제

# Part 2: 게시판

## 1. 개요

사내 공지사항, 부서별 정보 공유, 자유 토론 등을 위한 게시판 시스템이다.

---

## 2. 기능 상세

### 2.1 게시판 관리 (관리자)

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| BRD-001 | 게시판 생성 (이름, 설명, 유형) | 필수 |
| BRD-002 | 게시판 수정/삭제 | 필수 |
| BRD-003 | 게시판 유형: 공지사항, 일반, 부서별 | 필수 |
| BRD-004 | 게시판별 접근 권한 설정 (읽기, 쓰기, 관리) | 필수 |
| BRD-005 | 부서별 게시판 자동 생성 | 중요 |
| BRD-006 | 게시판 정렬 순서 설정 | 중요 |
| BRD-007 | 게시판 카테고리 관리 | 선택 |
| BRD-008 | 익명 게시 가능 여부 설정 | 선택 |

### 2.2 게시글

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| POST-001 | 게시글 작성 (제목, 본문, 카테고리) | 필수 |
| POST-002 | 본문 서식 에디터 (볼드, 이탤릭, 표, 이미지, 링크) | 필수 |
| POST-003 | 첨부파일 (최대 5개, 개별 20MB) | 필수 |
| POST-004 | 게시글 수정/삭제 (작성자, 관리자) | 필수 |
| POST-005 | 공지 고정 (게시판 상단 고정, 관리자) | 필수 |
| POST-006 | 필독 게시글 지정 및 읽음 확인 추적 | 중요 |
| POST-007 | 조회수 표시 | 필수 |
| POST-008 | 게시글 목록 페이지네이션 (20개/페이지) | 필수 |
| POST-009 | 게시글 검색 (제목, 내용, 작성자) | 필수 |
| POST-010 | 기간별 필터 | 중요 |
| POST-011 | 새 게시글 알림 (공지사항) | 중요 |

### 2.3 댓글

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| CMT-001 | 댓글 작성 | 필수 |
| CMT-002 | 대댓글 (1뎁스) | 중요 |
| CMT-003 | 댓글 수정/삭제 (작성자, 관리자) | 필수 |
| CMT-004 | 내 게시글에 새 댓글 시 알림 | 중요 |
| CMT-005 | 댓글 수 표시 | 필수 |

### 2.4 필독 게시글

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| MUST-001 | 게시글을 "필독"으로 지정 (관리자) | 중요 |
| MUST-002 | 대상자에게 필독 알림 발송 | 중요 |
| MUST-003 | 읽음/안읽음 현황 추적 (관리자 조회) | 중요 |
| MUST-004 | 미확인 필독 게시글 뱃지 표시 | 중요 |

---

## 3. 기본 게시판 구성

| 게시판 | 유형 | 읽기 권한 | 쓰기 권한 | 설명 |
|--------|------|----------|----------|------|
| 공지사항 | notice | 전체 | 관리자 | 전사 공지 |
| 자유게시판 | general | 전체 | 전체 | 자유 주제 |
| 건의사항 | general | 전체 | 전체 | 건의/제안 (익명 가능) |
| {부서명} 게시판 | department | 해당 부서 | 해당 부서 | 부서별 자동 생성 |

---

## 4. 비즈니스 규칙

1. 삭제된 게시글의 댓글도 함께 삭제된다
2. 공지 고정 게시글은 최대 5개까지 가능하다
3. 필독 게시글은 읽음 확인 버튼을 클릭해야 읽음 처리된다 (단순 조회 X)
4. 게시판 삭제 시 소속 게시글이 있으면 경고 후 확인 필요
5. 부서 게시판은 해당 부서원과 상위 관리자만 접근 가능
6. 게시글 작성자가 퇴직해도 게시글은 유지 ("퇴직한 사용자" 표시)
