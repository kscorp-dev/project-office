# 모바일 dev/preview/production 빌드 가이드

마지막 갱신: 2026-04-30 (v0.20.0)

EAS Build 로 iOS/Android 실기기 빌드를 만들고, CallKit/CallKeep + Expo Push 통합이
실제로 동작하는지 단계별로 검증하는 가이드.

## 0. 사전 점검 (자동)

```bash
cd apps/mobile
npm run verify:build
```

36개 항목을 점검 — 한 개라도 ✗ 이면 EAS 빌드 전에 고쳐야 한다.
주요 항목:
- app.json: bundleIdentifier / version
- iOS UIBackgroundModes (voip / audio / remote-notification)
- iOS 권한 설명 7종 (App Store 리젝션 방지)
- Android 권한 (CallKeep 9종 + WebRTC 4종)
- config plugin 4종 (callkeep / webrtc / notifications / aps-environment)
- eas.json 프로파일별 EXPO_PUBLIC_API_URL + APS_ENV
- usePushNotifications.ts 카테고리 3종 (approval / message / meeting)

## 1. 빌드 프로파일 매트릭스

| 프로파일 | 배포 | API | APNs | 용도 |
|---------|------|------|------|------|
| development | internal | https (43-200-...) | development | 사내 테스트, dev client |
| development-simulator | internal | localhost:3000 | development | 시뮬레이터, 노트북 백엔드 |
| preview | internal | https | production | TestFlight 배포 직전 검증 |
| production | store | https | production | App Store / Google Play |

```bash
npm run build:dev:ios      # 개발 iOS (실기기)
npm run build:dev:android  # 개발 Android (실기기)
npm run build:preview      # TestFlight 후보
npm run build:prod         # 프로덕션 릴리스
```

## 2. CallKit 동작 검증 (iOS 실기기 전용)

CallKit 은 iOS 시뮬레이터에서 **잠금화면 UI 가 재현되지 않는다**. 검증은 반드시 실기기.

### 2-1. 포그라운드에서 호출 트리거 (앱이 화면에 떠 있을 때)

1. 호스트 계정으로 로그인
2. 회의 생성 → 시작 → "📞 호출" 버튼 누르기
3. 다른 기기(참가자) 의 앱이 **포그라운드** 상태에서 CallKit 풀스크린 UI 출현 확인
4. [수락] 버튼 → /meeting/:id/room 으로 진입
5. [거절] 버튼 → CallKit UI 닫힘 + 호스트에게 "통화 거절" 알림 도착

### 2-2. 잠금화면에서 인라인 [수락]/[거절] (앱이 백그라운드)

1. 참가자 기기 잠금
2. 호스트가 /ring 호출
3. 잠금화면에 알림 도착 + [수락] [거절] 버튼 노출 확인
4. 길게 누르면 두 버튼 표시되어야 함 (3D Touch / Long Press)
5. [수락] 누르면 앱 부팅 + /meeting/:id 자동 이동

### 2-3. 회의 초대 (스케줄)

1. 회의를 미래 시각으로 예약
2. 참가자에게 일반 알림 도착 (CallKit 풀스크린 UI 는 안 뜸 — meta.ring 없음)
3. 알림 본체 탭 → /meeting/:id 진입

### 2-4. CallKit UI 가 안 뜰 때 디버그

증상별:
- "알림은 오는데 CallKit UI 가 안 뜸" → app.json `UIBackgroundModes.voip` 누락
- "알림 자체가 안 옴" → entitlements.aps-environment 가 development 인데 TestFlight 배포 / 또는 그 반대
- "포그라운드에선 안 뜸" → addNotificationReceivedListener 가 호출 안 됨 (로그 확인)
- "수락/거절 버튼이 안 보임" → setNotificationCategoryAsync('meeting') 실패 (iOS 14 미만 지원 X)

## 3. Expo Push 토큰 + 백엔드 등록

빌드된 실기기에서:
1. 앱 부팅 후 콘솔에서 `[push] register failed` 로그 없어야 함
2. 백엔드에서 `prisma.userDevice.findMany({ where: { isActive: true } })` 결과 확인
3. 토큰이 `ExponentPushToken[xxxxx]` 형식이어야 함

## 4. APNs 환경 분기 (production vs development)

`with-aps-environment.js` 플러그인이 `EXPO_PUBLIC_APS_ENV` 환경변수를 보고
`Project Office.entitlements` 의 `aps-environment` 키를 빌드 시점에 분기 설정.

```
development 빌드 → aps-environment = development → APNs sandbox 토큰
production 빌드  → aps-environment = production  → APNs production 토큰
```

토큰을 잘못된 환경으로 발송하면 silent fail 한다.
`expo-server-sdk` 는 자동으로 적절한 엔드포인트를 선택하므로 백엔드 코드는 변경 불필요.

## 5. EXPO_ACCESS_TOKEN (백엔드 측)

`apps/backend/.env`:
```
EXPO_ACCESS_TOKEN=<expo dashboard 에서 발급>
```

미설정 시:
- 푸시 발송은 동작하지만 rate limit 이 매우 낮음 (분당 ~600개)
- 프로덕션에서는 반드시 설정

`DISABLE_PUSH=true` 는 테스트 / CI 전용. 운영에선 미설정.

## 6. 일반적인 빌드 실패 패턴

| 에러 | 원인 | 해결 |
|-----|------|------|
| `consteval` Pods/fmt | Xcode 16.3+ Clang 20+ | `with-fmt-xcode26-fix.js` (이미 적용) |
| CallKeep 모듈 not found | EAS Build 캐시 | `eas build --clear-cache` |
| `aps-environment` mismatch | 잘못된 entitlement | EXPO_PUBLIC_APS_ENV 확인 |
| iOS 빌드 시 BIND_TELECOM 거절 | iOS 에 해당 권한 없음 | Android 전용 권한이라 무시 |
